# 自動提交工作流程

當偵測到程式碼變更時，自動執行以下步驟：

## 1. 檢查變更狀態
```bash
git status --porcelain
```

## 2. 自動添加變更檔案
```bash
git add .
```

## 3. 分析變更內容並生成提交訊息
- 檢查修改的檔案類型
- 根據檔案內容變更自動生成適當的提交訊息
- 包含時間戳記和變更摘要

## 4. 自動提交
```bash
git commit -m "自動提交: [變更摘要] - $(date '+%Y-%m-%d %H:%M:%S')"
```

## 5. 自動推送到遠端
```bash
git push origin main
```

## 使用方式
此工作流程會在以下情況自動觸發：
- 編輯器偵測到檔案儲存
- 程式碼內容發生變更
- 經過設定時間間隔（預設 5 分鐘）

## 注意事項
- 只會提交有實際變更的檔案
- 自動跳過暫存區和設定檔
- 保持提交歷史清晰可讀
